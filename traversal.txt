## Root-Level Traversal (All Subfolders Are Tickets)

* The script operates from a root directory containing multiple subfolders.

* Each immediate subfolder is treated as an independent processing root (ticket folder).

* For each subfolder:
  * Use it as the starting point for traversal.
  * Apply the traversal rules defined below inside this folder.

* Ignore any files in the root directory — only process directories.

* Ensure:
  * Traversal does not cross between subfolders
  * Each subfolder is processed independently
  * All collected files across all subfolders are copied into a single flat `./temp` directory
  * Filename conflicts are handled globally using incremental suffixes:
    * file.jpg
    * file_1.jpg
    * file_2.jpg

* If an error occurs while processing a subfolder:
  * Log the error
  * Continue with the next subfolder


---

## Traversal Rules (Applied Inside Each Subfolder)

* Starting point:
  * The traversal begins at the root of the current subfolder (ticket folder).

* Recursively find all source folders matching:
  * `Master Files`
  * `Deliverables`
  * Matching is case-insensitive and ignores spaces/special characters.

---

### When a Source Folder is Found

* For each matching source folder:

  * Copy all supported files found at the current directory level.

  * Inspect only immediate child folders.

---

### Version Folder Detection

* If immediate child folders contain version folders:

  * Extract version numbers from folder names.
  * Select the folder with the highest version.
  * Ignore all other version folders.

  * Continue traversal only inside the selected version folder.
  * Mark traversal as "inside version branch".

---

### No Version Folder Found

* If no version folders exist among the immediate children:

  * Recurse into all child folders.
  * Continue applying the same traversal rules.

---

### Behaviour Inside a Version Branch

* Once inside a version branch:

  * Continue collecting supported files at each level.
  * Recurse into all child folders.

  * Do NOT perform version-folder detection again.
  * Treat all folders as normal folders.

---


## File Collection Rules

* For each processed subfolder (ticket folder):
  * Create a dedicated output directory:
    * `./output/<folderName>/`

  * All files collected during traversal of that subfolder must be copied into its corresponding output directory.

---


### Structure Rules

* Do not preserve the original source folder structure.
* Each ticket folder must produce a flat structure inside its own output directory.

Example:

output/
  ├── TicketP1/
  │     ├── file.jpg
  │     ├── image.png
  │     ├── video.mp4
  │
  ├── TicketP2/
        ├── file.jpg
        ├── file_1.jpg
